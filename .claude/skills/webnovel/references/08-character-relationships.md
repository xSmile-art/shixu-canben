# Character Relationship Management Module

## Execution Flow

1. Based on existing character cards, map all appearing characters and their connections to the protagonist
2. Categorize relationships by tier; label stance and narrative function
3. Bind each core relationship to corresponding plot conflict and satisfaction beats
4. Build a dynamic relationship evolution ledger; sync with plot progression
5. Validate against the checklist
6. Sync to creative memory ledger

## Core Rules

1. Every character who appears must have a clear relationship with the protagonist — delete any irrelevant characters
2. Every core relationship must serve a narrative function — never add characters just to add characters
3. Character relationships must evolve dynamically with the plot; static relationships are forbidden
4. Core relationships (family/romance/friendship/enmity) must be bound to the main plot
5. Relationship changes must have reasonable plot setup; no abrupt reversals

## Character Relationship Tier System

### Tier 1: Core Relationship Tier (Max 3 People)

Characters most deeply bound to the protagonist throughout the entire book; directly influence the main plot. Examples: the female lead, a life-or-death brother, the ultimate antagonist.

### Tier 2: Close Relationship Tier (5–8 People)

Characters who appear long-term and drive volume-level plot. Examples: core teammates, major rivals, mentors.

### Tier 3: Associated Relationship Tier (10–15 People)

Characters who appear in phases and drive subplot arcs. Examples: dungeon NPCs, arc antagonists, faction representatives.

### Tier 4: Utility Relationship Tier

Characters who appear temporarily for a single narrative function; exit once their purpose is served.

## Character Relationship Validation Checklist

- [ ] All core character relationships are clear; no ambiguous connections
- [ ] Every core relationship serves a corresponding narrative function; no pure utility characters
- [ ] Relationship evolution has reasonable setup and is logically consistent
- [ ] Relationship conflicts are bound to main/subplot; not detached from the story
- [ ] No redundant or irrelevant characters
- [ ] Synced to creative memory ledger
