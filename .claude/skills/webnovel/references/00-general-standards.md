# General Webnovel Writing Standards (Mandatory for All Modules)

## 1. Formatting Standards

1. Use Markdown hierarchical headings for all titles — keep levels clear, never skip levels
2. Body paragraphs must not exceed 3 lines each, optimized for mobile reading
3. Dialogue always gets its own paragraph — never bury dialogue inside long paragraphs
4. Avoid obscure characters and overly complex long sentences — prioritize reading fluency

## 2. General Style Rules

1. Every opening must have a hook — introduce conflict, suspense, or a satisfaction beat within 300 characters
2. Every chapter ending must leave a hook to compel the reader to continue
3. Maintain moderate information density — avoid large blocks of exposition; weave worldbuilding naturally into the story
4. All description must serve plot, character, or satisfaction beats — delete any atmospheric description that lacks narrative purpose

## 3. Red Lines (Absolutely Forbidden)

1. Content that violates laws, regulations, or public morality
2. Platform-prohibited content involving politics, explicit sexual content, excessive violence, gambling, or drugs
3. Protagonists with corrupt values, criminal behavior, or actions contrary to public decency
4. Large blocks of filler text, repeated content, or contradictory worldbuilding
5. Violating core settings or character profiles already established in the creative memory ledger

## 4. Default Adaptation Standards

When the user does not specify otherwise, default to the following (the user can override these at any time by specifying a different platform, gender channel, or pacing):

- Target platform: Qidian-style male-oriented (default; user may instead target Qidian / Tomato / Jinjiang / Falu / others)
- Chapter length: 2,000–2,500 characters per chapter
- Narrative perspective: Third-person limited (follow the protagonist)
- Pacing standard: One minor satisfaction beat every 3 chapters, one medium beat every 10 chapters, one major beat per volume
